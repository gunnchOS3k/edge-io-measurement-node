# Problem → Solution Map — Edge-IO Measurement Node

## The human problem
Twins without real device experience misrepresent latency and outages communities actually feel.

Who is harmed: Students and residents if measurements leak PII or mislead planners.

## The technical problem
Schema-validated, consent-gated telemetry export to 7GC.

## The research gap
Existing tools rarely combine **equity**, **open reproducibility**, and **cross-repo evidence** for under-connected communities at Gary-scale fidelity.

## This repo's solution
Telemetry schema, privacy helpers, demo/smoke generator, 7GC export contract.

## What runs today
`make smoke` → `results/e2e/ telemetry export sample`

## What the output means
Smoke-test artifact for CI and portfolio review.

## What the output does NOT prove
Representative city-wide measurement campaign.

## How a researcher can extend it
Pick one `next_evidence` item; document benchmark or field protocol; link PR to `[Evidence TODO]` issue.

## How a WAIKE learner can contribute
Run smoke test · file reproduction issue · fix docs/tests · pair with mentor on one evidence issue.
